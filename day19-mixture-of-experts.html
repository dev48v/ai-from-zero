🧠 Mixture of Experts — Big Models, Cheap Inference

Watch a router send each token to only a few experts — huge model, tiny per-token compute

This layer holds 8 expert sub-networks plus a small router. Feed a sentence through one word at a time: the router scores all 8 experts for that token and routes it to the top-2. Only those two light up and do work — the other six stay dark (sparse activation). Flip to Dense mode and all 8 fire for every token (full compute). Same layer, a quarter of the work per token.

Only the routed experts do math → ≈ a quarter of the FLOPs of the dense layer.

Instead of every token using the whole network, a router sends each token to a few specialized experts. So a model can hold hundreds of billions of parameters (huge capacity) yet only activate a small slice per token (cheap compute) — and over a sentence, different tokens flow to different experts, so experts quietly specialize. This demo uses hand-tuned routing so each token's path is deterministic; real routers learn their scores during training, and routing is far noisier.

🔨 Build a tiny Mixture-of-Experts layer

    
A MoE layer is surprisingly small in code: a list of ordinary expert networks, one linear router that scores them, a top-k pick, a weighted combine of the chosen experts' outputs, and an auxiliary load-balancing loss so the router spreads work out instead of overusing a couple of favorites. Below is a minimal PyTorch-style sketch you can paste and grow — the same shape used by production sparse models such as Mixtral (8 experts, top_k=2).
