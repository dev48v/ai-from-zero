🔨 Get real embeddings

    
You don't need to train anything. An embedding model converts any text to a vector in one API call.

    
    
      
  1. 
        
    
          
    1
    
          
    Pick an embedding model
    
        
    
        
    Any major AI provider has one. They all do the same job: text in → vector of numbers out. Common choices:
    
        
    
          
    text-embedding-3-small (OpenAI) — 1,536 dims, cheap, great quality
    
          
    all-MiniLM-L6-v2 (HuggingFace) — free, runs locally, 384 dims
    
          
    embed-english-v3.0 (Cohere) — strong for retrieval tasks
    
          
    nomic-embed-text (Ollama) — fully offline, no API key needed
    
        
    
      
    2. 
      
  2. 
        
    
          
    2
    
          
    One API call = one vector
    
        
    
        
    Here's all the code you need (Python). The response is just a list of floats.
    

    from openai import OpenAI
client = OpenAI()

resp = client.embeddings.create(
    model="text-embedding-3-small",
    input="The quick brown fox"
)
vector = resp.data[0].embedding
print(len(vector))   # → 1536
print(vector[:4])    # → [0.021, -0.043, 0.009, 0.067, ...]

    Compare two pieces of text
    
        
    
        
    Cosine similarity between two vectors tells you how related they are. 1.0 = identical meaning, 0 = unrelated, negative = opposite.
    

    import numpy as np

def cosine(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

v1 = embed("dog")
v2 = embed("puppy")
v3 = embed("cloud")

print(cosine(v1, v2))  # → ~0.91  (very similar)
print(cosine(v1, v3))  # → ~0.18  (unrelated)
