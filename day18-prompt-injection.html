A model can't inherently tell your instructions from data — if untrusted text reaches the prompt, it may be obeyed. Indirect injection is the same trick hidden inside a web page, document, or email the model is asked to read. Defense is about isolation (delimit / structure untrusted data), screening inputs, least privilege for any tools, and never trusting model output blindly. This demo uses hand-written logic so each case is deterministic; real attacks and defenses are far messier.

🔨 Defend an app against prompt injection (real patterns)

    
There is no single fix for prompt injection — you stack defenses so that if one fails, another still holds. Below is a practical, layered playbook you can apply to any app that puts untrusted text into a prompt: isolate the data, screen it going in, validate what comes out, lock down any tools the model can call, keep secrets out of the prompt entirely, and test relentlessly with known attack strings. Copy each pattern as a starting point.
