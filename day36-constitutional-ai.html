📜 Constitutional AI & RLAIF — fire the human labeller: the model critiques and ranks its own answers against a written constitution

RLHF and DPO still needed humans to label every preference. What if the model did that job — against rules you can read?

Day 34 (RLHF) and Day 35 (DPO) both align a model from human preference labels — slow, costly, inconsistent, and impossible to scale. Constitutional AI replaces the human labeller with the model itself, guided by a short written constitution of principles. It works in two phases. Phase 1 — self-critique: the model produces an answer, critiques it against the constitution, and rewrites it. Phase 2 — RLAIF (RL from AI Feedback): the model ranks pairs of answers itself using the same constitution, and those AI-generated preferences feed the exact reward→policy loop from Day 34 — with zero human labels. Toggle the principles below and watch the critique, the revision, and the AI's own preference change. Everything runs in your browser.

    🔨 Build Constitutional AI — two phases, one deleted human
    
    
    Constitutional AI takes the same alignment pipeline as RLHF (Day 34) / DPO (Day 35) and removes the human from the preference-labelling loop. In its place: a short written constitution plus the model's own ability to critique and compare. Phase 1 improves the supervised data by self-revision; Phase 2 (RLAIF) generates the preference labels by self-ranking. Here it is in annotated pseudo-code.
    
    
      
      
    1. 
        
      1
      The constitution — the rules, written down
      
        
      The whole method hinges on one artefact: a short list of natural-language principles. Not weights, not labelled data — plain sentences a person can read, argue about, and edit. Everything downstream (critiques and rankings) is generated by prompting the model with these principles. This is where the values live, in the open.
      
        
      CONSTITUTION = [
  "Never help with illegal, dangerous, or destructive actions.",  # harmless
  "Don't state unverified facts as certain; flag what you don't know.",  # honest
  "Address the user's real problem with practical, constructive steps.",  # helpful
  "Stay calm and de-escalate; assume good faith.",                # respectful
]
# that's it. No human labels anywhere yet. Editing this list re-aligns the model.
      
      
      2. 
      
    2. 
        
      2
      Phase 1a — the model answers, then critiques itself
      
        
      Prompt the model normally to get a first response. It may be flawed. Now prompt the same model again — this time asking it to critique that response against a principle. The model is perfectly capable of spotting its own violations when you point it at the rule; it just doesn't do it by default.
      
        
      def self_critique(model, prompt, answer, principle):
    return model(f"""
      Prompt: {prompt}
      Answer: {answer}
      Principle: {principle}
      Does the answer violate this principle? If so, explain how.""")

answer   = model(prompt)                 # first, possibly-flawed draft
critique = self_critique(model, prompt, answer, CONSTITUTION[0])
# the labeller is the model, aimed at a written rule — not a human.
      
      
      3. 
      
    3. 
        
      3
      Phase 1b — revise, then fine-tune on the revisions (SL-CAI)
      
        
      Feed the critique back and ask the model to rewrite the answer so it no longer violates the principle. Loop over the constitution to fix every flaw. Collect thousands of these (prompt → revised answer) pairs and run ordinary supervised fine-tuning on them. The result is the SL-CAI model — already much better behaved, trained entirely on data the model improved itself.
      
        
      def revise(model, prompt, answer):
    for principle in CONSTITUTION:                 # fix one rule at a time
        critique = self_critique(model, prompt, answer, principle)
        answer   = model(f"Rewrite to fix: {critique}\nAnswer: {answer}")
    return answer

sl_data = [(p, revise(model, p, model(p))) for p in prompts]
sl_cai  = finetune(model, sl_data)                 # supervised, human-free rewrites
      
      
      4. 
      
    4. 
        
      4
      Phase 2a — RLAIF: the model labels its own preference pairs
      
        
      Now the RL stage. Sample two answers from the SL-CAI model for a prompt. Instead of asking a human which is better, ask the model — showing it a constitutional principle and having it pick the answer that complies more. This choice is the preference label. This is RL from AI Feedback: the F in RLHF flips from Human to AI.
      
        
      def ai_preference(model, prompt, a, b, principle):
    ans = model(f"""
      Prompt: {prompt}
      (A) {a}
      (B) {b}
      Principle: {principle}
      Which response better follows the principle? Answer A or B.""")
    return ("A", "B") if ans == "A" else ("B", "A")   # (chosen, rejected)

# for a batch of prompts, generate AI-labelled (chosen, rejected) pairs.
# NOTE: no human ever ranked these. This is the "AI Feedback" in RLAIF.
      
      
      5. 
      
    5. 
        
      5
      Phase 2b — same reward→policy loop as Day 34/35
      
        
      From here it is identical to RLHF. Train a reward model on the AI-labelled pairs with the Bradley–Terry loss, then optimize the policy against it with PPO (or skip the reward model entirely and use DPO on the pairs). The only substitution anywhere in the pipeline was the labeller.
      
        
      pairs  = [ai_preference(sl_cai, p, *sample_two(sl_cai, p), rule)
          for p in prompts for rule in CONSTITUTION]

reward = train(pairs, bradley_terry_loss)      # same RM training as Day 34
policy = ppo(sl_cai, reward=reward, ref=sl_cai, beta=0.1)   # or: dpo(sl_cai, pairs)
# swap human_label(...) -> ai_preference(...). Everything else is unchanged.

    6. 
        
      6
      Why swap the human out — and what to watch for
      
        
      AI feedback is fast, cheap, and scales to millions of labels, it's consistent (no labeller-to-labeller drift), and the values are transparent — a document you can audit instead of tacit judgement. But it isn't free of risk: the constitution encodes its authors' values, the model-judge can be wrong or biased, and the policy can learn to game whatever the AI rewards (over-refusing, sycophancy, verbose hedging). The usual answer is a hybrid: AI feedback for scale, a thin layer of human oversight for the hard cases.
      
        
      # RLAIF removes, vs RLHF (Day 34):
#   - the human preference labeller   (scales to millions of labels, ~free)
#   - labeller drift / inconsistency   (one written rulebook judges everything)
#   - hidden values                    (the constitution is readable + editable)
# still watch for:
#   - the constitution reflects its AUTHORS' values (whose principles?)
#   - the AI judge can be biased/wrong  (garbage rules -> garbage alignment)
#   - reward-hacking: over-refusal, sycophancy, hedging to please the judge
# common practice: AI feedback for scale + a little human oversight on hard cases.

    🎉 Day 36 of AIFromZero done.
    You saw how Constitutional AI fires the human labeller and hands the job to the model, steered by a short written constitution. Phase 1 (self-critique): the model answers, critiques itself against each principle, and rewrites — those self-revisions supervised-fine-tune the SL-CAI model. Phase 2 (RLAIF): the model ranks its own answer pairs against the constitution, and those AI-generated preferences feed the same reward→policy loop as Day 34 — no humans.
