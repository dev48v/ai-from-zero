🤖 AI Agents — an LLM That Can Use Tools and Act

Give the model tools and a loop — and it stops just talking

A chatbot answers in one shot. An agent runs in a loop: Thought → Action (call a tool) → Observation (the result) → repeat, until it has what it needs. Watch one solve a task.

🛠️ Tools available: calculator, search, get_hours. The model decides WHICH to call and WHEN, reads the result, and loops until it can answer.

🔨 The agent loop, in code

    
An agent is mostly a while-loop around the model. ~15 lines of orchestration.

    
    
      
  1. 1
    Describe the tools to the model
    
        
    const tools = {
  calculator: (expr) => eval(expr),
  search:     (q)    => searchWeb(q),
  get_hours:  (biz)  => db.hours(biz),
};
// the model is told each tool's name, purpose, and arguments
    2. 
      
  2. 2
    Loop: let it think, then act
    
        
    while (true) {
  const step = await llm(history);          // model emits Thought + Action
  if (step.type === "answer") return step.text;
  const result = tools[step.tool](step.args); // run the tool
  history += `Observation: ${result}`;        // feed the result back
}
    3. 
      
  3. 3
    The observation closes the loop
    Each tool result is appended to the context, so the next Thought is informed by what actually happened — not a guess. That feedback is what makes an agent grounded instead of hallucinating.
    4. 
      
  4. 4
    Guardrails matter
    Cap the iterations (so it can't loop forever), validate tool inputs, and gate risky actions (sending email, spending money) behind approval. Autonomy needs limits.
