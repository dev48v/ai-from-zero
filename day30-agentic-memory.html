🧩 Agentic Memory — How an Agent Remembers Across Steps & Sessions

The tool-using agent from Day 28-29 forgets everything the moment its context fills up — or the session resets. Memory is what fixes that.

An agent has two tiers of memory. Working memory is the context window — fast but tiny and volatile; when it fills, older turns get summarised/compacted to make room, and when the session ends it's gone. Long-term memory is a small vector store the agent writes salient facts to and later retrieves by similarity — it survives across sessions. Send the agent messages ("my name is …", "I prefer …"), watch it decide what's worth remembering, watch working memory fill and compact, then hit 🔄 New session to wipe the context and ask "what's my name?" — the answer comes back from long-term. Every write, embed, cosine-retrieval and compaction below is real code running in your browser.

🔨 Give a tool-using agent a memory

    
Memory is two stores wired around the model loop: a token-capped working buffer that holds the live context, and a long-term vector store the agent writes salient facts to and retrieves from by similarity. Here's the whole thing, layer by layer.

    
    
      
  1. 
        
    1
    Define the two stores
    
        
    Working memory is a list capped by token_budget — it is the context window. Long-term memory is a vector store: each record keeps the text, its embedding, a type, and a timestamp. Working memory is fast and volatile; long-term is durable and searchable.
    
        
    class Agent:
    def __init__(self, token_budget=4000):
        self.working = []          # recent turns — the live context window
        self.budget  = token_budget
        self.longterm = VectorStore()   # {text, vector, type, ts} — persists

# types: "semantic" (distilled facts), "episodic" (events),
#        "procedural" (rules / how-to notes)
    
      
    2. 
      
  2. 
        
    2
    Embed & retrieve by similarity
    
        
    Long-term recall is just RAG (Day 15) over the agent's own history. Embed every stored fact once; at query time embed the query and return the top-k by cosine similarity. Real systems use a proper embedding model + ANN index — the shape is identical.
    
        
    def retrieve(store, query, k=4):
    q = embed(query)
    scored = [(cosine(q, m.vector), m) for m in store]
    scored.sort(reverse=True)             # most similar first
    return [m for score, m in scored[:k] if score > 0]
    
      
    3. 
      
  3. 
        
    3
    A write policy — decide what's worth keeping
    
        
    You don't store every message — that's just a transcript. Extract the salient bits: stable facts (name, preferences) as semantic, events as episodic, standing instructions as procedural. Skip questions and chit-chat. Cheap regex works; better to let a small model distill "what should I remember from this?"
    
        
    def write_policy(text):
    facts = []
    if m := re.search(r"my name is (\w+)", text, re.I):
        facts.append(("semantic",  f"User's name is {m[1]}"))
    if m := re.search(r"i (prefer|like) (.+)", text, re.I):
        facts.append(("semantic",  f"User prefers {m[2]}"))
    if re.search(r"\b(always|never|remember to)\b", text, re.I):
        facts.append(("procedural", text))
    return facts        # nothing salient? store nothing.
    
      
    4. 
      
  4. 
        
    4
    Assemble the prompt every turn
    
        
    On each turn: write new facts to long-term, retrieve the relevant ones, and build the context as system prompt + running summary + retrieved memories + recent turns. Retrieval is what makes a fact from 500 turns ago show up exactly when it's needed.
    
        
    def step(agent, user_msg):
    for typ, fact in write_policy(user_msg):
        agent.longterm.add(typ, fact)          # WRITE salient facts
    hits = retrieve(agent.longterm, user_msg)  # READ what's relevant
    prompt = [SYSTEM, agent.summary, *hits, *agent.working, user_msg]
    reply = model(prompt)
    agent.working += [user_msg, reply]
    compact(agent)                             # keep it under budget
    return reply

  5. 
        
    5
    Compact on overflow — summarise the oldest turns
    
        
    The context window is finite. When working memory exceeds its budget, fold the oldest turns into a short running summary and keep only the most recent turns verbatim. This is reflection/compaction — the same move a long chat does automatically. Distilled facts should already be safe in long-term, so compaction loses phrasing, not knowledge.
    
        
    def compact(agent):
    while count_tokens(agent.working) > agent.budget:
        old = agent.working[:-KEEP_RECENT]     # everything but last few
        agent.summary = summarise(agent.summary, old)  # LLM or extractive
        agent.working = agent.working[-KEEP_RECENT:]   # keep recent verbatim

  6. 
        
    6
    Forgetting, conflicts & frameworks
    
        
    Long-term memory must decay or it rots. De-duplicate, let newer facts supersede stale ones ("name is now Bob"), and rank by relevance × recency so the current truth wins. In practice you reach for LangGraph checkpoints, MemGPT / Letta (the OS-style paging that inspired this), or mem0 — and you honour privacy: memory is user data, so redact PII and support "forget me".
    
        
    def remember(store, typ, fact):
    dup = retrieve(store, fact, k=1)
    if dup and cosine(embed(fact), dup[0].vector) > 0.95:
        store.update(dup[0], fact)     # supersede a near-duplicate
    else:
        store.add(typ, fact, ts=now())
    store.evict(older_than=TTL, keep_top=CAP)   # decay / forget
    # frameworks: LangGraph · MemGPT/Letta · mem0   (don't hand-roll)
