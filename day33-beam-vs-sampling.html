🎲 Beam Search vs Sampling — how a model turns "probabilities per token" into actual words

A model doesn't hand you a sentence — at every step it hands you a probability distribution over the next token. Decoding is the rule you use to turn that distribution into one chosen word, over and over.

Same model, same prompt "the movie was …", five different decoding rules. Greedy grabs the single most-likely token every step (and gets stuck in a loop). Beam search carries the top-B whole sequences ranked by cumulative log-probability and returns the best (great for translation, oddly bland for open text). Temperature, top-k, and top-p (nucleus) reshape or trim the distribution and then roll the dice — trading determinism for diversity. Pick a strategy, tune its knobs, and watch it generate token-by-token. Every distribution, log-prob, beam prune, and sampled draw below is real code running in your browser on a tiny scripted next-token model.

🔨 Build every decoder from one distribution

    
A decoder is a thin wrapper around the one thing the model already gives you: logits → a probability distribution over the vocabulary, at every step. Greedy, beam, temperature, top-k and top-p are each a handful of lines on top of that. Here's the whole ladder, from the simplest rule to the sampler a modern chat model actually ships with.

    
    
      
  1. 
        
    1
    Start from the distribution the model already produces
    
        
    Every step the model emits logits — one raw score per vocabulary token. Softmax turns them into a probability distribution that sums to 1. Decoding is just: given this distribution, which token do I emit, and how do I use its shape? Everything below operates on probs.
    
        
    import torch, torch.nn.functional as F
logits = model(input_ids).logits[:, -1, :]   # scores for the NEXT token
probs  = F.softmax(logits, dim=-1)            # distribution over the vocab, sums to 1
# probs = {"the":0.32, "very":0.30, "really":0.16, ...}
    
      
    2. 
      
  2. 
        
    2
    Greedy = argmax (and watch it loop)
    
        
    The simplest rule: take the single highest-probability token every step. Fully deterministic, dirt cheap — and it frequently walks straight into a repetition loop ("very very very…") because the locally-best token isn't the globally-best sequence. Fine for short, factual completions; poor for anything open-ended.
    
        
    def greedy(model, ids, max_new=40):
    for _ in range(max_new):
        probs = F.softmax(model(ids).logits[:, -1, :], dim=-1)
        nxt = probs.argmax(dim=-1, keepdim=True)   # THE most-likely token
        ids = torch.cat([ids, nxt], dim=-1)
        if nxt.item() == EOS: break
    return ids
# deterministic: same input -> same output, every time
    
      
    3. 
      
  3. 
        
    3
    Beam search = carry the top-B sequences by cumulative log-prob
    
        
    Instead of committing to one token, keep B partial sequences ("beams"). Each step: expand every beam over the vocabulary, score each candidate by its cumulative log-probability (sum of log-probs — you add in log space to avoid underflow), then prune back to the top B. Return the best complete beam. It searches wider than greedy and shines on translation/summarization where there's one "right" answer.
    
        
    def beam_search(model, ids, B=4, max_new=40):
    beams = [(ids, 0.0)]                       # (sequence, cumulative logprob)
    for _ in range(max_new):
        cand = []

        beams = sorted(cand, key=lambda x: x[1], reverse=True)[:B]   # prune to top-B
    return max(beams, key=lambda x: x[1])[0]   # best cumulative logprob wins
    
      
    4. 
      
  4. 
        
    4
    Temperature — reshape the distribution before you sample
    
        
    Divide the logits by T before softmax. T<1 sharpens the peak (more confident, closer to greedy); T>1 flattens it (higher entropy, more surprising, more diverse). T→0 is greedy; T→∞ is uniform noise. This is the "creativity dial" — it changes how random the draw is, before any trimming.
    
        
    def sample_temperature(logits, T=0.8):
    probs = F.softmax(logits / T, dim=-1)      # /T reshapes: <1 sharper, >1 flatter
    return torch.multinomial(probs, num_samples=1)   # roll the dice
# T=0.0 -> greedy   |   T=1.0 -> the model's own probs   |   T=2.0 -> wild
    
      
    5. 
      
  5. 
        
    5
    Top-k and top-p — trim the tail, then sample
    
        
    Pure temperature sampling can still, rarely, pick a garbage token from the long tail. So we cut the tail first. Top-k keeps a fixed number k of the most-likely tokens. Top-p / nucleus keeps the smallest set whose probabilities sum to ≥ p — an adaptive cutoff that's wide when the model is unsure and narrow when it's confident. Renormalize the survivors, then sample.
    
        
    def top_p(probs, p=0.9):
    s, idx = torch.sort(probs, descending=True)
    cum = torch.cumsum(s, dim=-1)
    keep = cum - s < p            # smallest set whose cumulative prob >= p
    s[~keep] = 0.0
    s = s / s.sum()               # renormalize the "nucleus"
    return idx[torch.multinomial(s, 1)]
# top-k is the same idea with a FIXED cutoff: keep s[:k], zero the rest
    
      
    6. 
      
  6. 
        
    6
    The real sampler: temperature + top-p + a repetition penalty
    
        
    Modern chat models combine them: apply a repetition/presence penalty to logits of tokens already used, scale by temperature, trim with top-p (often plus a top-k safety cap), then sample. Rule of thumb: need one correct answer? greedy or low temperature (even beam for MT). Need variety, brainstorming, natural chat? temperature ≈ 0.7–1.0 with top-p ≈ 0.9. Set temperature = 0 when you need reproducibility.
    
        
    def decode_step(logits, prev_ids, T=0.8, p=0.9, rep=1.15):
    for t in set(prev_ids):
        logits[t] /= rep                 # 1) discourage repeats
    logits = logits / T                  # 2) temperature
    probs  = F.softmax(logits, dim=-1)
    probs  = top_p(probs, p)             # 3) nucleus cutoff (+ optional top-k)
    return torch.multinomial(probs, 1)   # 4) sample
# task -> deterministic (greedy / T=0 / beam);  chat -> sample (T~0.8, p~0.9)

🎉 Day 33 of AIFromZero done.
You saw that a language model never outputs a sentence — it outputs a distribution per step, and decoding is the rule that turns those distributions into words. Greedy takes the argmax (fast, deterministic, loops). Beam search keeps the top-B sequences by cumulative log-probability and returns the best (excellent for translation, generic for open text, still repeats). For open-ended and chat generation we sample: temperature reshapes the distribution, top-k keeps a fixed cutoff, and top-p / nucleus keeps an adaptive one — the modern default being temperature + top-p, often with a repetition penalty.
