AIFromZero · Day 41

✨ Emergent Abilities — some skills sit near zero in small models, then appear abruptly once you cross a scale threshold: multi-digit arithmetic, multi-step reasoning, following instructions. The twist: much of that "sudden" jump is an artifact of a harsh, all-or-nothing metric (exact-match). Score the same model with a continuous metric and the very same capability rises smoothly — emergence is partly in the ruler, not the model (Schaeffer et al.).

Abilities that "switch on" — real phase change, or a trick of how we measure?

Day 40 showed loss falls as a smooth, forecastable power law. But downstream skills can behave differently: on many tasks a model scores essentially 0% across a huge range of sizes, then — past some threshold — accuracy leaps. That's an emergent ability (Wei et al. 2022), and it's unsettling: you can't read the jump off the smooth loss curve, which matters for both forecasting and safety. Then came the counter-argument (Schaeffer et al. 2023, "Are Emergent Abilities a Mirage?"): the jump is largely manufactured by a discontinuous metric — exact-match, all-or-nothing. Both demos below compute their curves from one smooth latent-skill function, live in your browser, no model calls.

② The mirage — one skill, two rulers — flip the metric: harsh exact-match = cliff · continuous metric = smooth, from the SAME latent skill

Fix one multi-step task. Its latent per-step skill s(x) is a single smooth sigmoid — no cliff anywhere. Exact-match scores it as sn (all n steps right): raising a number below 1 to a big power stays ~0 until s is nearly perfect, then shoots up — an apparent emergent jump. Flip to a continuous metric — token accuracy, edit-distance similarity, or average log-prob — and the same capability rises gradually and forecastably. Slide the task length n and watch the "threshold" move even though the model never changed.

Two views, one function. Card ① shows why abilities look like they switch on at a scale threshold — flat near 0, then a cliff — and why harder/longer tasks emerge later. Card ② reveals the catch: score that same smooth capability with a continuous metric and the cliff dissolves into a gentle S-curve. Some "emergence" is a real phase change worth watching; a lot of it is a mirage of the all-or-nothing ruler. The safe read: report a continuous metric too, don't assume you can forecast capability from loss alone — and still watch for genuinely abrupt jumps.

🔨 Build an emergence-vs-mirage simulator

    
Reproduce the whole phenomenon from one smooth latent-skill function. Score it with a harsh all-or-nothing metric to manufacture an "emergent" cliff, then with continuous metrics to reveal the same capability rising smoothly — the exact engine behind LOOK ① and ②.

    
    
      
  1. 
        
    1
    One smooth latent skill — no cliff anywhere
    
        
    Start with a single, gently-rising probability that the model gets one step (one token / one reasoning move) right. It's a plain sigmoid in log-scale — perfectly smooth. Everything else is derived from this.
    
        
    import numpy as np
# x = log10(model size or compute). ONE smooth per-step skill:
def skill(x, x0=9.2, k=1.55):
    return 1 / (1 + np.exp(-k * (x - x0)))   # s(x) in (0, 1), NO cliff
# x0 = the scale where a single step is 50% likely to be correct.

  2. 
        
    2
    The harsh metric — exact-match manufactures a cliff
    
        
    A task counts as solved only if all n steps are correct. That's s ** n. Because s is below 1, raising it to a large power keeps the score pinned near zero until s is nearly perfect — then it rockets up. That knee is the "emergent ability".
    
        
    def exact_match(x, x0, n):
    return skill(x, x0) ** n        # all n steps must be right
# s rises GENTLY, but s**n stays ~0 until s is close to 1,
# then SHOOTS up -> a sharp cliff that looks like emergence.
# bigger n (longer task) => the cliff appears at a LARGER scale.

  3. 
        
    3
    Continuous metrics — the same skill, smoothly
    
        
    Now score the identical model with metrics that give partial credit. Token accuracy, edit-distance similarity and average log-prob all read s directly (or a monotone function of it) — so they inherit its smooth, forecastable shape. No cliff.
    
        
    def token_accuracy(x, x0):  return skill(x, x0)          # fraction of steps right
def edit_similarity(x, x0): return skill(x, x0)          # 1 - norm. edit distance ~ s
def avg_log_prob(x, x0):    return np.log(skill(x, x0))  # continuous, concave, smooth
# all three rise gradually with scale -> capability is predictable.

    Reproduce the mirage — one model, two curves
    
        
    Sweep scale once. Plot exact-match and token accuracy on the same axis. The only difference between the cliff and the smooth curve is the metric — the underlying s(x) is identical. That is the core of Schaeffer et al.'s argument.
    
        
    xs = np.linspace(6, 12, 200)              # 1M ... 1T params
cliff  = exact_match(xs, x0=9.2, n=25)    # flat ~0, then SUDDEN jump
smooth = token_accuracy(xs, x0=9.2)       # gentle S-curve, SAME model
# same s(x) underneath -> the metric alone creates the "emergence".

  5. 
        
    5
    Show the threshold is an artifact of n
    
        
    Find where exact-match crosses 50%. Invert the sigmoid at the skill level s = 0.5 ** (1/n). Increase n and the apparent threshold slides to larger scale — even though the latent skill never moved. The "phase transition" lives in the ruler.
    
        
    def apparent_threshold(x0, n, k=1.55):
    s = 0.5 ** (1/n)                  # skill needed so that s**n = 0.5
    return x0 + np.log(s/(1-s)) / k   # invert the sigmoid -> log10(params)

apparent_threshold(9.2,  5)   # short task  -> emerges earlier
apparent_threshold(9.2, 40)   # long  task  -> emerges much later
# crank n up: threshold moves RIGHT, but skill(x) is unchanged.

    The honest takeaway — and why it's a safety issue
    
        
    Not all emergence is a mirage: genuine sharp transitions may exist, and users often do care about all-or-nothing outcomes (a proof either checks out or it doesn't). So measure deliberately — and keep watching for real jumps, because a dangerous capability appearing abruptly is exactly what you can't afford to miss.
    
        
    # practical guardrails:
# 1) always report a CONTINUOUS metric alongside exact-match
#    (token acc / edit dist / log-prob / Brier) -> it forecasts.
# 2) smooth loss (Day 40) does NOT imply smooth capability.
# 3) exact-match matters when the USER needs the whole answer right.
# 4) for safety: keep monitoring for ABRUPT new abilities
#    (deception, tool misuse) as you scale -> real jumps can hide.

Some abilities look like they switch on at a scale threshold — flat near 0, then a sudden cliff (Wei et al. 2022). But score the same smooth latent skill with an all-or-nothing metric and you get sn — a manufactured cliff; swap to a continuous metric and the identical capability rises gently and forecastably (Schaeffer et al. 2023). So a lot of "emergence" is a mirage of the ruler — yet real phase changes may still lurk, which is why it matters for capability forecasting and safety.
