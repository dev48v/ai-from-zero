AIFromZero · Day 42

🌐 World Models — an agent that learns a simulator of its own environment: feed it a state + an action and it predicts the next state. Now the agent can plan and "dream" thousands of rollouts inside the model — cheap, fast, safe — instead of paying to act in the slow, costly, risky real world (Ha & Schmidhuber's World Models; Dreamer). The catch: the model is imperfect, so feeding its own predictions back in makes long imagined rollouts drift as errors compound.

Dream inside a learned model — and watch the imagined world drift from the real one

Here is a tiny 2-D world: a ball bouncing in a box under gravity, nudged by actions. Its true dynamics are a real hand-coded physics step — (state, action) → next state — the "expensive real world". Beside it lives a learned world model: the same equations but with imperfect parameters (its estimate of gravity, bounciness, drag). Roll the model forward from a start state, feeding its own predictions back in (a "dream" / imagined rollout), and compare it step-by-step to the true trajectory. Small errors compound — the dream drifts, and a bounce can blow the gap wide open. Everything below is computed live in your browser: real state-transition math, real drift metric. No neural net, no server.

🔎 Same model, longer dream = more drift. This is the core limit of learning in imagination: short rollouts stay grounded and are gold for planning & sample-efficiency; long ones hallucinate a world of their own. Real systems fight back with uncertainty/ensembles, periodic re-grounding in real data, and rollouts in a compact latent space — not raw pixels.

A world model lets an agent plan by dreaming — cheap, fast, safe rollouts instead of costly real-world trials, the engine behind Ha & Schmidhuber's World Models, Dreamer, MuZero-style latent planning, and why video-generation models are called implicit world models. But every prediction feeds the next, so an imperfect model's errors compound and the dream drifts. The craft of world models is buying as many trustworthy imagined steps as you can before reality and imagination part ways.

🔨 Build a world model & dream inside it

    
Reproduce the whole LOOK demo from scratch: a real hand-coded environment, a deliberately-imperfect learned model with the same shape, an autoregressive rollout that feeds predictions back in, and a drift metric that reveals compounding error. This is the exact engine powering LOOK ① and ②.

    
    
      
  1. 
        
    1
    State, action, and the TRUE environment dynamics
    
        
    A world is just a function (state, action) → next state. Here the state is a bouncing ball {x, y, vx, vy} in a unit box, the action is a velocity nudge, and the "true" step is real physics: apply the action, gravity, drag, integrate, then reflect off the walls with restitution. This is the slow, costly real world an agent would normally have to act in.
    
        
    const R = 0.022, DT = 1;                       // ball radius, one tick per step
const TRUE = { g:0.0013, e:0.88, drag:0.9985, imp:0.010, vbias:0 };
const ACT = a => [{ax:0,ay:0},{ax:-1,ay:0},{ax:1,ay:0},{ax:0,ay:1}][a];

function physStep(s, a, p){                     // ONE parameterised physics step
  let {x,y,vx,vy} = s; const {ax,ay} = ACT(a);
  vx += ax*p.imp;  vy += ay*p.imp;              // action = velocity impulse
  vy -= p.g;  vx += p.vbias;                    // gravity (+ systematic bias)
  vx *= p.drag; vy *= p.drag;                   // drag
  x += vx*DT;  y += vy*DT;                       // integrate
  if (x<R){x=R;vx=-vx*p.e;} if (x>1-R){x=1-R;vx=-vx*p.e;}   // wall bounce
  if (y<R){y=R;vy=-vy*p.e;} if (y>1-R){y=1-R;vy=-vy*p.e;}   // (restitution e)
  return {x,y,vx,vy};
}
const trueStep = (s,a) => physStep(s, a, TRUE);

    2
    The learned world model — same shape, imperfect parameters
    
        
    A trained world model never recovers the true physics exactly — it has a slightly wrong idea of gravity, bounciness, drag and how much an action does. We model that honestly: the same physStep, but with parameters offset by an err knob. At err = 0 the model is perfect (it equals the true env); crank it up and it's confidently wrong.
    
        
    function modelParams(err){                       // err in [0,1]
  return {
    g:    TRUE.g    * (1 + 0.45*err),            // thinks gravity is stronger
    e:    TRUE.e    * (1 - 0.18*err),            // thinks walls are less bouncy
    drag: TRUE.drag - 0.004*err,                 // thinks there's more drag
    imp:  TRUE.imp  * (1 - 0.35*err),            // underestimates its own actions
    vbias: 0.0018*err                            // a small systematic sideways bias
  };
}
const modelStep = (s,a,err) => physStep(s, a, modelParams(err));
// err=0  =>  modelParams === TRUE  =>  dream == reality (no drift).
    
      
    3. 
      
  3. 
        
    3
    The rollout — step reality vs dream, side by side
    
        
    Give both the same start state and the same action sequence. Step the true env with trueStep. "Dream" with modelStep — and crucially feed the model its own previous prediction (si), never the true state. That autoregressive feedback is exactly how an agent imagines a future it hasn't lived, and exactly why errors get a chance to snowball.
    
        
    function rollout(s0, actions, err, N){
  const trueT=[s0], imagT=[s0], errs=[0];
  let st=s0, si=s0;                              // st = reality, si = imagination
  for (let k=0;k<N;k++){
    const a = actions[k];
    st = trueStep(st, a);                        // what really happens
    si = modelStep(si, a, err);                  // dream: prediction feeds itself
    trueT.push(st); imagT.push(si);
    errs.push(Math.hypot(st.x-si.x, st.y-si.y)); // drift at this step
  }
  return { trueT, imagT, errs };
}

    4
    The drift metric — quantify how far the dream has wandered
    
        
    A single number per step: the Euclidean distance between imagined and true position. From the whole errs array we read the final drift, the mean drift, and the reliable horizon — the last step before the dream crosses a tolerance. That horizon is the honest answer to "how many steps can I plan on?".
    
        
    function metrics(errs, tol=0.06){
  const N = errs.length - 1;
  const finalDrift = errs[N];
  const meanDrift  = errs.reduce((a,b)=>a+b,0) / errs.length;
  let horizon = N;                               // first step that breaks tolerance
  for (let k=1;k<=N;k++) if (errs[k] > tol){ horizon = k-1; break; }
  return { finalDrift, meanDrift, horizon };
}
// horizon = how long you can TRUST the imagined world before it lies.

    Why the error compounds (and bounces amplify it)
    
        
    Each step's tiny bias shifts the state the model reads next, so errors are fed back in and grow roughly geometrically. Worse, near a wall a small position error flips whether the ball has bounced yet — after which the two trajectories head opposite ways. That's why longer dreams don't just get a bit worse, they can diverge hard.
    
        
    const s0 = { x:0.3, y:0.75, vx:0.02, vy:0.0 };
const acts = Array.from({length:90}, ()=>0);     // "coast" — just gravity + bounces

const short = rollout(s0, acts.slice(0,10), 0.35, 10);
const long  = rollout(s0, acts,             0.35, 90);
metrics(short.errs).finalDrift;   // small — dream still ~ reality
metrics(long.errs).finalDrift;    // much larger — errors compounded + a bounce hit
// same model, same err: the ONLY change is how far ahead we dreamed.

    Learning in imagination — the payoff, and the guardrails
    
        
    The whole point: train a controller on dreamed experience instead of costly real steps (Ha & Schmidhuber's V-M-C, Dreamer). It's wildly sample-efficient — but only if you respect the horizon. Keep rollouts short, re-ground in real data, and use uncertainty to know when to stop trusting the dream.

🎉 Day 42 of AIFromZero done.
A world model is a learned (state, action) → next state simulator of an environment. With it an agent can plan and learn by dreaming — rolling out imagined futures cheaply, quickly and safely instead of paying to act in the real world (Ha & Schmidhuber's Vision-Memory-Controller; Dreamer; and why video models like Sora are read as implicit world models, plus latent planning in MuZero/PlaNet). The price is compounding prediction error: because each prediction feeds the next, an imperfect model's dream drifts from reality over long horizons — so short rollouts, re-grounding and uncertainty are what keep imagination useful. 👉 Tomorrow — Day 43: Grokking — when a model overfits for ages, then suddenly generalizes long after the training loss flatlined.
