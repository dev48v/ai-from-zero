👍 RLHF — teaching a model to behave: SFT → reward model → PPO with a KL leash

Pretraining gives a model knowledge. RLHF gives it manners. It happens in three stages — and the last one is on a leash.

A base LLM just predicts the next token from the internet — it doesn't know it's supposed to be helpful, honest, or harmless. RLHF (Reinforcement Learning from Human Feedback) aligns it in three moves: (1) SFT imitates human-written demonstrations; (2) humans rank pairs of answers and a reward model learns to score them; (3) PPO optimizes the model against that reward — while a KL penalty keeps it from drifting off the reference model and gaming the score. Step through each stage below. Every reward score, gradient step, and reward-vs-KL trade-off is real code running in your browser.

Stage 3 · Optimize the reward — but stay on the leash

PPO treats the model as a policy and pushes it to produce answers the reward model scores highly. Left unchecked it will happily drift far from the reference model and start gaming the reward. The KL penalty is the leash: it charges the policy for every step away from the frozen reference (the SFT model). Set the leash length, then run PPO and watch the trade-off.

🔨 Build the RLHF pipeline, stage by stage

    
RLHF is three training jobs chained together: a supervised fine-tune, a reward model trained on human preference pairs, and a PPO loop that optimizes the policy against that reward with a KL penalty back to the reference. Here's the whole thing in annotated pseudo-code.

    
    
      
  1. 
        
    1
    Stage 1 — SFT: imitate human demonstrations
    
        
    Start from a pretrained base model. Collect a dataset of prompt → ideal-answer pairs written by humans, and fine-tune with ordinary next-token cross-entropy. The result follows instructions and is the reference policy everything else is measured against.
    
        
    base   = load_pretrained("base-llm")          # knows language, not manners
demos  = [(prompt, human_written_answer), ...]  # a few thousand demonstrations

sft = finetune(base, demos, loss="cross_entropy")   # plain supervised learning
# sft now follows instructions. FREEZE a copy as the reference:
ref = freeze(copy(sft))                        # PPO's KL penalty anchors here
    
      
    2. 
      
  2. 
        
    2
    Collect preference data — rank, don't write
    
        
    Writing perfect answers is expensive; judging two answers is cheap. For each prompt, sample a few completions from the SFT model and have humans pick the better one. Each label is a (chosen, rejected) pair — far more data per dollar, and it captures preferences you could never fully write down.
    
        
    prefs = []
for prompt in prompts:
    a, b = sft.sample(prompt), sft.sample(prompt)   # two candidate answers
    chosen, rejected = human_rank(prompt, a, b)     # label: which is better
    prefs.append((prompt, chosen, rejected))
# prefs = a dataset of comparisons, not gold answers
    
      
    3. 
      
  3. 
        
    3
    Train the reward model (Bradley–Terry)
    
        
    The reward model r(x, y) outputs a single scalar for a prompt/answer. Train it so the chosen answer scores higher than the rejected one. The Bradley–Terry loss says: the probability a human prefers the chosen answer is σ(r_chosen − r_rejected) — maximize it.
    
        
    reward_model = add_scalar_head(copy(sft))   # LLM body + a 1-number output head

def rm_loss(x, y_chosen, y_rejected):
    r_c = reward_model(x, y_chosen)
    r_r = reward_model(x, y_rejected)
    return -log_sigmoid(r_c - r_r)          # push chosen above rejected

train(reward_model, prefs, loss=rm_loss)    # now r(x,y) ~ human preference
    
      
    4. 
      
  4. 
        
    4
    The RLHF objective — reward minus a KL leash
    
        
    Now optimize the policy to maximize reward. But the reward model is only a proxy — push too hard and the policy finds gibberish that scores high but is useless (reward hacking). So subtract a KL-divergence penalty from the frozen reference: every token of drift costs reward. β sets the leash length.
    
        
    # per-token reward given to PPO:
#   reward_model score, minus how far the policy strayed from the reference
def rlhf_reward(x, y):
    r  = reward_model(x, y)                       # want this HIGH
    kl = log pi_theta(y|x) - log pi_ref(y|x)      # per-token KL to reference
    return r - beta * kl                          # the leash: penalize drift
# beta large  -> short leash, stays near ref (safe, smaller gains)
# beta small  -> long leash,  higher proxy reward but risks drift / hacking
    
      
    5. 
      
  5. 
        
    5
    Stage 3 — the PPO loop
    
        
    PPO (Proximal Policy Optimization) is the RL algorithm that applies that reward. Each iteration: roll out answers from the current policy, score them with the reward-minus-KL signal, estimate advantages, and take a clipped gradient step so the policy never moves too far at once. Repeat until reward plateaus.
    
        
    policy = copy(sft)                              # start the policy AT the reference
for step in range(n_ppo_steps):
    prompts = sample_batch()
    ys   = policy.generate(prompts)             # 1) roll out answers
    rew  = rlhf_reward(prompts, ys)             # 2) reward - beta*KL
    adv  = advantages(rew, value_fn)            # 3) how much better than expected
    # 4) clipped policy-gradient update (the "proximal" part = small, safe steps)
    ratio = policy.logp(ys) / policy_old.logp(ys)
    loss  = -min(ratio*adv, clip(ratio, 1-eps, 1+eps)*adv).mean()
    policy.update(loss)
return policy                                    # the aligned model
    
      
    6. 
      
  6. 
        
    6
    Watch for over-optimization — proxy ≠ truth
    
        
    The single most important RLHF failure mode: as you optimize harder, proxy reward keeps rising while true quality peaks and then falls. The reward model was trained on limited data; off-distribution the policy exploits its blind spots. The KL leash and early stopping keep you in the region where the proxy still tracks reality.
    
        
    # proxy reward (RM) rises monotonically with training...
# ...but GOLD (true human) quality rises, peaks, then DROPS -> over-optimization
if kl_to_reference > kl_budget:        # drifted too far off-distribution
    stop_or_raise_beta()               # the RM is no longer trustworthy here
# modern shortcut: DPO skips the separate RM + PPO loop, optimizing
# the preference pairs directly against the reference (Day 35).

🎉 Day 34 of AIFromZero done.
You saw the three moves that turn a raw next-token predictor into a helpful assistant. SFT fine-tunes the base model on human demonstrations, giving it instruction-following and a reference policy. A reward model then learns from cheap human preference rankings (Bradley–Terry: push the chosen answer's score above the rejected one) to produce a scalar reward. Finally PPO optimizes the policy against that reward — but subtracts a KL penalty to the frozen reference so it can't drift off-distribution and reward-hack the proxy. The whole game is reward up vs KL up: pull too gently and you gain nothing, pull too hard and true quality collapses even as proxy reward climbs. The KL leash keeps you in the sweet spot. 👉 Tomorrow — Day 35: DPO (Direct Preference Optimization) — skip the reward model and PPO entirely and train straight on preference pairs.
