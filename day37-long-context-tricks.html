🧵 Long-Context Tricks — attention costs n², so how do models read a whole book? Windows, sparsity, sinks, RoPE scaling & the "lost in the middle" catch

Self-attention makes every token look at every other token. That's n² work — and it explodes.

Double the context and you quadruple the compute and memory. That single fact is why a 200k-token window is hard. The whole long-context playbook is a set of tricks to dodge that wall: only attend locally (sliding window), attend sparsely with a few global hops, keep a handful of attention sinks so streaming stays stable, interpolate positions (RoPE scaling) to stretch past the trained length — and then live with the fact that models get lost in the middle of a long prompt. ② Attention sinks · StreamingLLM — can a fixed window stream forever?

A rolling window seems perfect for endless chat — just drop old tokens. But softmax has to put its probability somewhere, and the very first tokens quietly become an "attention sink". Evict them and the scores blow up → perplexity explodes. Keep ~4 sink tokens and streaming stays stable forever.

③ Lost in the middle — a long window ≠ a well-used window

Even when a fact fits in the context, models retrieve it far better when it's near the start (primacy) or the end (recency). Bury the "needle" in the middle and accuracy sags into a U-shape — and it sags deeper as the context grows. Drag the needle and watch retrieval dip.

🔨 Build a sliding-window attention mask

    
The single most important long-context trick is turning the dense n×n attention mask into a thin band. Here it is in annotated pseudo-code — starting from the quadratic problem, adding the window, then sinks, sparsity, RoPE scaling, and a fix for lost-in-the-middle.

    
    
      
  1. 
        
    1
    The problem — full attention is n×n
    
        
    Every query token scores every past key token. The scores form an n×n matrix, so compute and memory grow with n². Double the context and you 4× the cost. That is the quadratic wall the rest of this page dismantles.
    
        
    scores = Q @ K.T                     # shape (n, n)  <-- the n×n blowup
scores = scores.masked_fill(~causal, -inf)   # no peeking at the future
attn   = softmax(scores, dim=-1) @ V # cost & memory ~ O(n²)
# context 4k -> 16M scores. context 128k -> 16.8B scores. it doesn't scale.
    
      
    2. 
      
  2. 
        
    2
    Sliding-window (local) attention — O(n·w)
    
        
    Let each token attend only to the last w keys — a diagonal band instead of a full triangle. Cost drops from O(n²) to O(n·w), which is linear in context. Information still travels far because layers stack: with L window layers the effective receptive field is about L·w.
    
        
    def sliding_window_mask(n, w):
    mask = zeros(n, n, dtype=bool)
    for i in range(n):
        lo = max(0, i - w + 1)
        mask[i, lo : i + 1] = True   # causal AND within the window
    return mask                      # ~ n*w True cells, not n²
# receptive field grows with DEPTH: L stacked window layers -> reach ~ L*w.
    
      
    3. 
      
  3. 
        
    3
    Attention sinks — make streaming stable (StreamingLLM)
    
        
    A pure rolling window breaks for endless streaming: once it slides past the first tokens, the model destabilises and perplexity explodes. The fix is tiny — always keep the first few tokens as global "attention sinks." Softmax needs somewhere to dump leftover probability; those early tokens are it.
    
        
    def streaming_mask(n, w, sink=4):
    mask = sliding_window_mask(n, w)
    mask[:, :sink] = True    # first `sink` keys stay visible to everyone
    return mask
# WHY: softmax must sum to 1; early tokens absorb the "spare" attention.
# evict them -> scores blow up. keep ~4 -> stream millions of tokens, stably.
    
      
    4. 
      
  4. 
        
    4
    Sparse / dilated attention — cheap long range
    
        
    Local windows miss distant tokens. Add a few strided (dilated) hops plus a handful of global tokens so information can jump across the sequence — while the pattern stays ~O(n·w). This is the Longformer / BigBird recipe: local + global + random/strided.
    
        
    def sparse_mask(n, w, stride=64, globals=(0,1,2,3)):
    mask = sliding_window_mask(n, w)         # local detail
    for i in range(n):
        for j in range(0, i + 1, stride):    # dilated long-range hops
            mask[i, j] = True
    mask[:, list(globals)] = True            # a few always-visible tokens
    return mask                              # still ~O(n*w), not O(n²)

    RoPE scaling — stretch past the trained length
    
        
    A model trained at 4k tokens breaks if you feed it position 30,000 — it never saw those rotary angles. Don't extrapolate; interpolate. Squeeze the new positions back into the range the model already knows (Position Interpolation). NTK-aware scaling and YaRN refine this; a little fine-tuning recovers quality.
    
        
    # trained at 4k, want 32k?  scale positions into the known range.
scale = train_len / target_len       # e.g. 4096 / 32768 = 1/8
pos   = arange(target_len) * scale   # positions now live in [0, 4096)
q, k  = apply_rope(q, k, pos)        # rotary embeddings on scaled positions
# extrapolation breaks; interpolation (PI / NTK / YaRN) holds. fine-tune a bit.

    Mitigate "lost in the middle"
    
        
    A big window doesn't guarantee the model uses the middle of it. Accuracy is U-shaped: strong at the start and end, weak in between. So place the most important evidence at the edges, retrieve fewer chunks (tighter top-k), and re-rank so the needle sits first or last.
    
        
    # accuracy is U-shaped in position -> exploit primacy & recency:
ordered = [most_relevant] + middling_chunks + [second_most_relevant]
#          ^ start (primacy)                        ^ end (recency)
# also: smaller top-k (less to get lost in) + re-rank the needle to an edge.
# "long context" is a capacity, not a guarantee that every token is read.
