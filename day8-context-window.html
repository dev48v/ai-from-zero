🪟 The Context Window — the Model's Short-Term Memory

Everything the model "remembers" is just text in a fixed-size box

An LLM has no memory between calls — it only sees the tokens you send, up to a hard limit (the context window). Keep chatting; when the box fills, the oldest messages fall off the edge and are forgotten.

↑ Greyed messages have scrolled OUT of the window — the model literally can't see them anymore. Ask it about something in a faded message and it has no idea. Bigger window = more it can hold at once (and more you pay).

🔨 Work with the window

    
Since it's finite, managing what's IN it is a real skill.

    
    
      
  1. 1
    Every call re-sends the whole history
    The model is stateless. A chat app resends the full conversation each turn — that's how it "remembers". When that exceeds the window, something has to give.
    2. 
      
  2. 2
    Summarise old turns
    Replace the distant history with a short LLM-written summary, keeping recent turns verbatim. Compresses many tokens into few without losing the gist.
    3. 
      
  3. 3
    Retrieve, don't stuff (RAG)
    For a big knowledge base, don't paste it all in — store it externally and fetch only the few relevant chunks per question. The window stays small and focused.
    4. 
      
  4. 4
    Mind the cost + the "lost in the middle"
    You pay per token in the window, and models attend best to the START and END — facts buried in the middle of a huge context can get overlooked. Bigger isn't automatically better.
