Search finds meaning — not just matching words

Day 13 turned text into vectors. Now: type a question. We embed it, then find the nearest vectors in a little library of docs using cosine similarity. Watch how vector search finds the right doc by meaning while plain keyword search misses it.

③ The index — every doc is a dot; the query lands among them, nearest neighbours circled

Grey dots = stored documents (the index). The indigo ★ = your query. The circled dots are its nearest neighbours — the top results.

A vector database stores embeddings and finds the ones nearest to your query's embedding — so search works on meaning, not exact words. This demo uses ~10 docs with tiny hand-authored vectors so it's deterministic; at scale a vector DB uses an ANN (approximate nearest-neighbour) index to stay fast over millions of vectors.
